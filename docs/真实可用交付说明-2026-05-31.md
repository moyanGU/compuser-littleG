## 2. 当前正确配置

如果使用云雾接口，当前项目里的正确配置语义应为：

```powershell
$env:COMPUSER_MODEL_PROVIDER='openai-compatible'
$env:COMPUSER_MODEL_NAME='gpt-5.4-mini'
$env:COMPUSER_MODEL_BASE_URL='https://yunwu.ai/v1/chat/completions'
$env:COMPUSER_MODEL_API_KEY='***'
```

注意：

- `COMPUSER_MODEL_PROVIDER` 不是模型名；本轮仍只支持 `demo` 与 `openai-compatible`
- `gpt-5.4-mini` 应放在 `COMPUSER_MODEL_NAME`
- `https://yunwu.ai/` 站点根地址不是当前客户端需要的 API 地址；必须使用完整 `v1/chat/completions` 路径

- 已修正 `phase4-live-smoke` / `phase5-scorecard` 中对多窗口模板的过严断言
- 修正点仅限发布证据 harness，不改变产品主链路语义
- 修正后，多窗口模板按真实语义允许在参与比较的合法窗口集合内路由，不再被错误判成 `verification_failed`
