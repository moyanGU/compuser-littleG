# 普通用户实机测试清单

## 打开后先看
1. 页面默认应先显示普通用户入口，不应先看到完整技能库、trace、上下文或治理细节。
2. 左侧应主要是最近任务，右侧只保留必要确认、停止和少量过程信息。
3. 首屏应直接能看到推荐能力卡片或模板入口。

## 任务结果
1. 成功任务要清楚显示 `处理中 -> 已完成 -> 最终结果`。
2. 失败任务要清楚显示 `失败原因 + 建议先看哪里`。
3. 需要确认时要清楚显示 `为什么停下 + 现在能点什么`。
4. 中断任务要明确显示 `已停止 / 已中断`，不能伪装成完成。

## 技能显示
1. 普通界面默认不展示完整技能清单。
2. 只展示少量人话能力入口。
3. 审计和高级技能治理只在高级入口里看。

## 验证收口
1. `/skills` 返回里普通前端只消费摘要口径。
2. capability 默认不会全量暴露。
3. `tools.search` 和 `capabilities.search` 都应先搜再用。

## 本地启动前检查
1. web-panel 普通用户入口是 `npm run web:panel:launcher`。
2. 本地面板默认端口是 `4317`。
3. 当前产品默认指向 `http://127.0.0.1:8010/mcp`，除非你显式覆盖环境变量。
4. 这份清单只对应当前 `compuser` 的 web-panel 产品面，不再引用旧 `gui_agent` 路径。

## 当前环境结论
1. `GET /product/support-matrix` 返回 `200`。
2. `GET /product/scorecard-summary` 返回 `200`。
3. `GET /product/governance` 返回 `200`。
4. `GET /session/local-web-panel-session/state` 返回默认会话状态。
5. `POST /session/task` 应接受任务并返回 `202`。
