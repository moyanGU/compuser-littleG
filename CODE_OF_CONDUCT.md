# Code of Conduct

## Our Standard

We want this project to be a respectful, focused, and constructive place to collaborate.

Examples of positive behavior include:

- being respectful in disagreement
- giving clear, actionable technical feedback
- assuming good intent while staying honest about problems
- keeping discussion scoped to the code, behavior, or design at hand

Examples of unacceptable behavior include:

- harassment, insults, or personal attacks
- discriminatory language or behavior
- deliberately disruptive conduct
- sharing private information without permission

## Project Expectations

In this repository, contributors are especially expected to:

- keep claims about behavior and support grounded in the real implementation
- avoid scope creep and unrelated refactors in focused changes
- describe the verification they actually ran
- respect current architecture and documentation boundaries

## Reporting

If you experience or witness unacceptable behavior, open a private channel with the repository owner if possible, or report it through the contact path associated with the repository owner account.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, issues, pull requests, or other contributions that violate this Code of Conduct.

Repeated or severe violations may result in temporary or permanent exclusion from collaboration in this repository.
